# SpecPine Theme — Palette & Style

Cohesive WarGames-style CRT identity that ties every output channel of the
SpecPine app together: Pager LOG screens, framebuffer splashes, ANSI/ASCII
glyph frames, and the repo-level promo art under `images/specpine/`.

## Sources of truth

- **`theme.sh`** — colour tokens and `LOG_*` wrappers used by all bash code.
- **`palette.md`** — this file. Human-readable spec.
- **`glyphs/*.txt`** — ASCII frames shown via `show_ansi <name>` from
  `funcs_main.sh`.
- **`splash.fb` + `boot_animation/frame_*.fb`** — pre-rendered raw RGB565
  framebuffer assets, generated by `scripts/generate_theme_fb.py`. Played
  by `bin/specpine_splash.py` (which falls back to inline drawing if the
  pre-rendered frames are absent).

## Palettes

### Primary — WarGames green CRT phosphor

| Role | LOG colour | Framebuffer RGB | Use |
|---|---|---|---|
| Title | `blue` | `(8, 12, 8)` background, `(0, 220, 100)` text | Section headers, "Press OK to Start" |
| Good / nominal | `green` | `(0, 220, 100)` | Sweep rows in noise floor, success messages |
| Warn | `yellow` | `(255, 180, 40)` | Borderline RF, paused state |
| Alert | `red` | `(220, 50, 60)` | Strong RF (≥ -50 dBm), errors, anomalies |
| Hint | `cyan` | `(60, 220, 220)` | Secondary text, key prompts |
| Brand | `magenta` | `(255, 100, 200)` | SpecPine identity touches (sparingly) |

### Variant — Hackers '95 cyan/magenta

For promo art only (`images/specpine/specpine-poster-cyan.png`). Replaces
the title with `(60, 220, 220)` cyan accent and shifts highlights toward
magenta. Not used on-device in v1.2.

## Type & line conventions

- **Width**: every LOG-bound frame ≤ 50 columns (Pager LOG cap).
- **Borders**: `┌─[ TITLE ]────┐` top, `│ … │` content, `└──────┘` bottom.
- **Prompt prefixes** (used in all LOG and PNG content):
  - `$ ` — user-input or command line; rendered white in PNG, default in LOG.
  - `> ` — output from SpecPine; rendered amber → green in PNG, theme `GOOD` in LOG.
  - `! ` — alert / anomaly; rendered red in PNG, theme `ALERT` in LOG.
- **Sweep rows** in the text waterfall carry a coded prefix (`R:` / `Y:` / `G:`)
  set by the renderer based on peak dBm; the bash wrapper strips the tag and
  routes to the matching `LOG` colour.

## Effects (PNG only)

- 3-px scanlines on the dark background to evoke a CRT.
- Light Gaussian-blur "phosphor glow" pass over the final image.
- Faint 20-px grid in `(0, 70, 30)` for the WarGames look-and-feel.
- 14-px outer border + corner notches in the accent colour.

## Not in scope

- A Pager-firmware-level `/lib/pager/themes/specpine/` directory. That
  would replace the system theme for ALL payloads, not just SpecPine.
  Tracked in `images/specpine/launcher-icon.md`.
- A CSS/web theme for the Pager web UI at `:1471/`. Out of payload reach.

## Authoring checklist

- New ASCII frame? Use the `┌─[ TITLE ]─┐` border and put it in
  `data/theme/glyphs/<name>.txt`. Wire via `show_ansi <name>`.
- New LOG output? Use the `LOG_GOOD` / `LOG_WARN` / `LOG_ALERT` / `LOG_HINT`
  wrappers from `theme.sh` so a future palette swap re-skins one line at a
  time, not the whole codebase.
- New framebuffer asset? Add a render call to `scripts/generate_theme_fb.py`
  and place the output in `data/theme/` (one-shot) or
  `data/theme/boot_animation/` (animated). Reference the same RGB tokens
  from `theme.sh` so PNG and on-device assets stay in lock-step.
